I wanted to note three Python projects that I made before I started developing websites. They are notable because they directly inspired some of the sites in my portfolio. Though these projects were made early in my personal development as a programmer and web developer, I am still satisfied with how they turned out, and they speak to my interest in creating dynamic, useful user experiences—even if a quiz about Super Mario Bros. is “useful” only in an entertaining sense!

DDRL Champions League vehicle database

  • Date: June 2022
  • Code: ddrl_champions_league_vehicles
  • Learning objectives: Expand my knowledge of the Tkinter GUI framework, as well as the Pygubu Designer program, which uses the tkinter module to helps construct user interfaces; practice N-tier architecture; practice using the sqlite3 module to create a small database.

The DDRL Champions League vehicle database (the league became the DFDRL only recently), was my first independently made Python project—that is, it was my first Python project made that wasn’t a class assignment. I had been introduced to N-tier programming architecture during the spring term, and quickly realized that working with data and presenting information in a useful way was something I really enjoyed doing. With this program I wanted to create something useful for the league, but as I became more and more enamored of web development, I knew that, rather than reworking this old program, I wanted to create something new and more ambitious—hence the new league website.

Super Mario Bros. character database

  • Date: June/July 2022
  • Code: super_mario_bros_characters
  • Learning objectives: Continue to expand my knowledge of N-tier architecture; learn the csv module and how to work with Commas Seperated Values files.

This project began as a one-day exercise in learning how to work with CSV files. Because I needed data in order to work with them, I begain creating data based on characters from Mario games over the years and, as tends to happen, I soon realized that I had much bigger plans for my “one-day” project. My idea was to create a program that was partly a database of any or all characters from Mario games, but also a way one could test one’s knowledge of the games: the database for the program starts out empty, and it is up to the user to create the data based on their knowledge: character name, what game that character first appeared in, and what year that game came out. Of course, if one wanted to look the information up as they went, that would be ok, too, and I even provided a link to a resource in order to double-check answers.

It is my hope to soon create a web-based version of this program using the Django framework.

Super Mario Bros. quiz

  • Date: July 2022
  • Code: super_mario_bros_quiz
  • Learning objectives: Challenge myself to create an object-oriented approach to a question-based program.

Earlier in the summer of 2022, I had wanted to make a program that had an interviewer (the program) asking an interviewee (the user) a series of questions that would change based on the user’s response. I wanted to mimic interacting with an intelligent being. It was not on my mind at that point to create an object-oriented approach to the program, but I subsequently wanted to challenge myself to use that approach. Rather than rewriting the program, I decided to make a new one: sticking with the theme of Super Mario from the character database I had recently completed, I created this Super Mario Bros. quiz. It is the direct inspiration for my first Django project.
